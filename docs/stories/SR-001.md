## User Story
En tant qu'utilisateur, je veux uploader un fichier `.sav` et voir la liste de mes sessions sauvegardées afin de choisir quelle base analyser.

## Critères d'acceptation
- [ ] POST `/api/saves` accepte un fichier `.sav` (multipart/form-data, max 50MB)
- [ ] Le fichier est décompressé (zlib, skip 4 bytes header) et le JSON est validé
- [ ] Une entrée `save_sessions` est créée avec filename, sessionName, playtime, timestamp, worldTime
- [ ] GET `/api/saves` retourne la liste des sessions triées par date d'upload décroissante
- [ ] DELETE `/api/saves/{id}` supprime la session et toutes les données associées (cascade)
- [ ] Réponse 400 si le fichier n'est pas un .sav valide
- [ ] Tests unitaires SaveController + tests intégration avec Testcontainers

## Notes techniques
